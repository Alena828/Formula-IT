Задание 1
На основе созданной концептуальной модели для предметной области Библиотека опишите реляционную модель БД. 
Результат представьте в виде таблицы пример, которой приведен в соответствующем файле 
1.	Осуществите выбор СУБД (в нашем случае – это СУБД PostgreSQL)
2.	Произведите преобразование логической модели в реляционную модель данных:
a.	Определите схему БД - набор именованных схем отношений 
3.	Для каждого отношения:
a.	задайте уникальное имя (используйте латиницу)
b.	проверьте атрибутивный состав отношений:
i.	При наличии многозначных атрибутов устраните многозначность путем введения нового отношения
ii.	Убедитесь, что все атрибуты атомарны
c.	определите первичный ключ (при необходимости введите суррогатный атрибут)
d.	выделите альтернативные ключи (при суррогатном PK их наличие обязательно!!!)
4.	Определите связи между отношениями:
a.	Задайте внешние ключи
b.	Определите кратность связи. 
c.	При наличии связей многие-ко-многим или категориальных связей выполните их разрешение
d.	Удалите избыточные связи при их наличии
e.	Проверьте обоснованность наличия связей один-к-одному
f.	Определите стратегию поддержки ссылочной целостности 
5.	Для каждого атрибута определите: 
a.	его имя (латиницей)
b.	тип данных (используя документацию, выберите подходящие поддерживаемые конкретной СУБД типы данных)
c.	правила валидации (домен)
d.	отношения к NULL-значениям
e.	уникальность
f.	значения по умолчанию (если необходимо)
